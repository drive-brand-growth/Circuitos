# MBA Education Assessment & Certification Structure
## Complementing CDAIO Technical Certification

---

## Assessment Philosophy

**CDAIO Assessments** (Primary - Technical Focus):
- Quizzes: Technical knowledge validation
- Deliverables: Practical AI/data artifacts
- Exams: Comprehensive technical mastery
- Grading: Accuracy, completeness, technical rigor

**MBA Assessments** (Complementary - Business Focus):
- Case analyses: Business judgment and strategic thinking
- Financial models: Quantitative rigor and business acumen
- Presentations: Executive communication
- Grading: Strategic insight, business viability, leadership

### PILLAR 2: Data-Driven Decision Making (10 hours)

#### Module 2.1: Executive Decision Science
**Assessment Type**: Decision Tree Analysis

**Assignment**: Build decision tree for major strategic choice
- **Decision**: Build vs. buy AI capability, cloud migration, or similar
- **Deliverable**: Decision model + 5-page analysis
- **Time**: 6-8 hours

**Required Elements**:
1. Decision framing (what decision, what alternatives)
2. Decision tree with all branches
3. Probability estimates with sources
4. Outcome values (financial and strategic)
5. Expected value calculations
6. Sensitivity analysis
7. Recommendation with rationale

**Example Decision**: Should we build an in-house data platform or buy Snowflake/Databricks?
```
Decision Tree:
- Build ($50M investment)
  ├─ Success (60% prob): NPV $80M
  ├─ Partial (30% prob): NPV $20M
  └─ Failure (10% prob): NPV -$40M
  Expected Value: $55M

- Buy ($30M over 5 years)
  ├─ Success (80% prob): NPV $60M
  ├─ Partial (15% prob): NPV $30M
  └─ Failure (5% prob): NPV -$20M
  Expected Value: $51M

- Hybrid ($40M investment)
  ├─ Success (70% prob): NPV $70M
  └─ Failure (30% prob): NPV $0M
  Expected Value: $49M

Recommendation: Build (highest EV), but consider risk tolerance
```

**Grading Rubric** (100 points):
- Decision framing (10 points)
- Tree structure completeness (20 points)
- Probability estimates (20 points)
- Value estimates (25 points)
- Sensitivity analysis (15 points)
- Recommendation quality (10 points)

**Passing Score**: 70%

#### Module 3.4: Data Platform Investment Strategy
**Assessment Type**: Platform Selection & ROI

**Assignment**: Evaluate data platforms and build business case
- **Options**: Compare 3 platforms (e.g., Snowflake, Databricks, BigQuery)
- **Deliverable**: Comparison + recommendation + 3-year ROI model
- **Time**: 8-10 hours

**Evaluation Criteria**:
1. Performance (query speed, concurrency)
2. Cost (storage, compute, egress)
3. Ease of use (SQL, integrations, UI)
4. Scalability (data volume, users)
5. Governance (security, compliance, lineage)
6. Ecosystem (partners, tools, support)

**ROI Model**:
```
Costs (3 years):
- Platform: $X (licenses, consumption)
- Migration: $Y (one-time)
- Training: $Z

Benefits (3 years):
- Time savings: $A (faster insights, self-service)
- Better decisions: $B (data-driven, accurate)
- Cost reduction: $C (vs. current state)

NPV = Benefits - Costs
ROI = (Benefits - Costs) / Costs × 100%
```

**Grading Rubric** (100 points):
- Platform evaluation depth (30 points)
- Comparison quality (25 points)
- ROI model rigor (30 points)
- Recommendation clarity (15 points)

**Passing Score**: 70%

## Assessment Administration

### Submission
- All assessments submitted via learning platform
- Automatic plagiarism checking
- Version control (track iterations)

### Grading
- **MBA Assessments**: Instructor graded within 7 days
- **Rubric-based**: Transparent criteria
- **Feedback**: Constructive comments on all submissions
- **Resubmission**: One resubmission allowed if <70%

### Academic Integrity
- Individual work required (no group projects)
- Proper citation of sources
- Original analysis and recommendations
